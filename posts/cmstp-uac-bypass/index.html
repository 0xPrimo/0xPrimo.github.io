Introduction

While i was looking for a way to bypass UAC (User Account Control) i came across a LOLBIN (Living of the land binary) called cmstp.exe that can be abused to bypass UAC and execute arbitrary commands through an auto-elevated COM interface.

Abusing CMSTP.EXE

Running cmstp.exe with the following config will spwan an elevated cmd.exe.

[version]
Signature=$chicago$

[DefaultInstall]
RunPreSetupCommands=RunPreSetupCommandsSection

[RunPreSetupCommandsSection]
C:\Windows\System32\cmd.exe

[Strings]
ServiceName="EVIL"
ShortSvcName="EVIL"

Hurdle: User Options Box

The issue is that the user must select All users option then click OK in order to spawn the elevated cmd.exe.

Analyzing CMSTP.EXE

As a starter I’ve attached x64dbg to cmstp.exe and examined the threads call stack.

As you can see cmstp.exe calls DialogBoxParamW to display the options box, this is a very useful information because after opening cmstp.exe on IDA we’ll just take a look where this API get referenced.

IDA found the debugging symbols of the binary, this will speed up the analysis.

DialogBoxParamW is been called from two functions, the one we’ll be interesting to analyze is GetInstallOptions.

i’ve created this graph to show the execution flow of GetInstallOptions.

As you can see from the graph, there is two ways to create an elevated COM interface. from user options box or the process is already elevated.

Abusing CMSTP.EXE

cmstp.exe uses a function called IsMemberOfGroup to check if the running process is elevated base on the process SID. we can patch the conditional jump instruction that check the result of the IsMemberOfGroup function.
